# Data Structures and Algorithms (DSA)

## 📌 Purpose of this Repository

This repository exists to help me achieve three core goals:
1. **Consolidated Revision:** A single source of truth to review optimized solutions, edge cases, and algorithmic patterns before coding interviews.
2. **Progress Tracking:** A visual log of my consistency, growth, and problem-solving patterns over time.
3. **Git Version Control:** Hands-on practice using industry-standard Git workflows to manage and refine my codebases.

## 🗂️ Repository Structure

## ⚡ Problem-Solving Approach

For every problem included in this repository, I focus on:
* **Time Complexity ($O(N)):** Optimizing from brute-force to the most efficient mathematical or data-driven runtime.
* **Space Complexity ($O(1) or $O(N)$):** Minimizing memory usage through in-place modifications where possible.
* **Code Readability:** Writing clean, modular, and well-commented code that is easy to follow.
